# Architecture

repo-history is two deterministic halves with an LLM sandwiched between them, communicating only through files on disk.

The deterministic engine is a Python package. `git.py` is a read-only data-source boundary: a `GitRepo` that shells out to git via subprocess argument lists (never shell strings), returning frozen dataclasses. `analysis/` holds the one deliberate extension point — the pipeline shape is fixed, but the analysis *method* is a plugin behind a registry; each method takes a shared `AnalysisContext` and returns a common `AnalysisResult` (summary + episodes + sections). Method #1, `mechanical`, uses no LLM: it filters noise, detects reverts, ranks rename-following hotspots, computes change-coupling, and groups commits into episodes.

`work.py` materializes episodes into secret-scrubbed Markdown bundles wrapped in an untrusted-data envelope; `models.py` are the Pydantic contracts the LLM fills in; `build.py` renders the durable artifacts; `state.py` tracks the analyzed head for incremental re-runs. The LLM half is a packaged Claude Code skill that reads bundles, grades each claim's basis (observed vs inferred), and abstains rather than fabricate.

The output deliberately leads with prescriptive, evidence-linked guardrails (the part a controlled study found actually helps a coding agent) and demotes the narrative timeline/architecture to a human-onboarding subdirectory.

## How it evolved

- **Scaffold the repo-history CLI**: Introduces the src/repo_history package layout with a thin typer CLI (plan/build/status subcommands, all stubbed to exit 1 with a 'not implemented' notice) and codifies the core split: a deterministic Python engine does the mechanical work, while the LLM orchestration lives in a separate Claude Code skill.
- **Add read-only git access layer**: Introduces src/repo_history/git.py as the sole git-access boundary: frozen dataclasses (Commit, FileChange, Tag) plus a GitRepo handle whose private _run shells out to git. Adds a tests/ suite with a shared conftest fixture that builds a realistic temp history.
- **Pluggable analysis engine, materialization, build, skill, and incremental state**: Introduced the analysis/ package splitting a fixed framework (base.py: Analyzer ABC, registry, and the AnalysisContext/AnalysisResult/Episode contracts) from swappable method plugins (mechanical.py) and method-agnostic helpers (util.py). The engine became two deterministic halves — plan/materialize (work.py) and build/render (build.py) — bracketing an LLM map/reduce step whose output is validated by Pydantic contracts (models.py). Added security.py (secret scrubbing), state.py (incremental head tracking via index.json), and skillfile.py plus a packaged SKILL.md. Downstream steps depend only on the Episode/AnalysisResult contract, never on how a method computed it.
- **Docs, self-analysis example, an MCP serve layer, and git arg-injection hardening**: Adds a serving layer without touching the pipeline: memory.py exposes RepoMemory, a read-only query layer over the built JSON artifacts (no LLM, no git, no network), and mcp_server.py is a thin FastMCP stdio adapter over it, run via `repo-history mcp`; the MCP SDK is an optional [mcp] extra so the core install stays lean. timeline.json gained per-episode paths so a file can be mapped back to its history. git.py's boundary is tightened with a _check_ref validator plus --end-of-options before every positional ref.
- **Security and correctness hardening of the untrusted-repo boundary**: No new modules; hardening lands inside existing ones. security.py's scrubber gains token shapes (JWT, Google, Stripe live, sk-/sk-ant-, github_pat, npm, SendGrid, connection-string credentials) with all patterns linear-time and the private-key scan length-bounded. git.py switches log records to NUL (%x00) separation with a greedy body parse and raw_diff streams under a read cap. work.py adds an untrusted-data notice, dynamically-sized backtick fences, episode-id validation, and a per-line char cap. base.py/build.py degrade cleanly on empty repos and corrupt manifests. A cleanup replaces hand-rolled _jsonable with dataclasses.asdict.
- **Reframe output to lead with evidence-graded guardrails**: GUARDRAILS.md becomes the flagship root artifact; DECISIONS.md and LANDMINES.md stay at root while TIMELINE/ARCHITECTURE/HOTSPOTS move under onboarding/; JSON mirrors stay flat at the root for the MCP server. Decision and Landmine models gain a `basis` field, and _write now creates parent subdirectories.
- **Teach the skill to grade trust and abstain**: SKILL.md analysis schema now requires a `basis` field on decisions and landmines, and the documented build output describes the flagship GUARDRAILS.md plus the onboarding/ narrative.
