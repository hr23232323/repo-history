First commit lays down the Python package: a typer/pydantic CLI skeleton with plan/build/status stubs, packaging (pyproject + uv.lock), README vision, and MIT license. Establishes the project as a deterministic engine that hands the fuzzy analysis to an LLM orchestrator.

## Add read-only git access layer

Adds GitRepo, a read-only wrapper over git that walks commits chronologically, reports per-file churn with rename detection, fetches raw diffs, and resolves tags. Backed by a synthetic fixture repo (rename, extension, revert, annotated tag) with pinned dates and identity for deterministic tests.

**Architecture:** Introduces src/repo_history/git.py as the sole git-access boundary: frozen dataclasses (Commit, FileChange, Tag) plus a GitRepo handle whose private _run shells out to git. Adds a tests/ suite with a shared conftest fixture that builds a realistic temp history.

## Pluggable analysis engine, materialization, build, skill, and incremental state

Built the core of repo-history: a fixed pipeline with a swappable analysis method behind an Analyzer registry, a first no-LLM 'mechanical' method, materialization of episodes into secret-scrubbed Markdown bundles, a build step that renders durable .repo-memory artifacts from validated LLM output, a packaged Claude Code skill that drives the LLM map/reduce half, and incremental state tracking so re-runs only process new history. This is the deterministic-engine + LLM-orchestration architecture the whole tool rests on.

**Architecture:** Introduced the analysis/ package splitting a fixed framework (base.py: Analyzer ABC, registry, and the AnalysisContext/AnalysisResult/Episode contracts) from swappable method plugins (mechanical.py) and method-agnostic helpers (util.py). The engine became two deterministic halves — plan/materialize (work.py) and build/render (build.py) — bracketing an LLM map/reduce step whose output is validated by Pydantic contracts (models.py). Added security.py (secret scrubbing), state.py (incremental head tracking via index.json), and skillfile.py plus a packaged SKILL.md. Downstream steps depend only on the Episode/AnalysisResult contract, never on how a method computed it.

## Docs, self-analysis example, an MCP serve layer, and git arg-injection hardening

Documented the tool (README, adding-a-method guide) and shipped an unedited self-analysis run under examples/, added a read-only MCP server that serves the built .repo-memory/ JSON to agents, and hardened the git boundary against argument injection through refs.

**Architecture:** Adds a serving layer without touching the pipeline: memory.py exposes RepoMemory, a read-only query layer over the built JSON artifacts (no LLM, no git, no network), and mcp_server.py is a thin FastMCP stdio adapter over it, run via `repo-history mcp`; the MCP SDK is an optional [mcp] extra so the core install stays lean. timeline.json gained per-episode paths so a file can be mapped back to its history. git.py's boundary is tightened with a _check_ref validator plus --end-of-options before every positional ref.

## Security and correctness hardening of the untrusted-repo boundary

Six commits hardening the tool's defining risk that content from a possibly-hostile repo flows through bundles into an agent and into committed .repo-memory/: broadened secret scrubbing, an explicit untrusted-data posture at the LLM boundary, fixes for a commit-separator crash and tag-boundary grouping, graceful degradation on empty repos and corrupt manifests, plus a simplification pass and tests moved onto the real render path.

**Architecture:** No new modules; hardening lands inside existing ones. security.py's scrubber gains token shapes (JWT, Google, Stripe live, sk-/sk-ant-, github_pat, npm, SendGrid, connection-string credentials) with all patterns linear-time and the private-key scan length-bounded. git.py switches log records to NUL (%x00) separation with a greedy body parse and raw_diff streams under a read cap. work.py adds an untrusted-data notice, dynamically-sized backtick fences, episode-id validation, and a per-line char cap. base.py/build.py degrade cleanly on empty repos and corrupt manifests. A cleanup replaces hand-rolled _jsonable with dataclasses.asdict.
