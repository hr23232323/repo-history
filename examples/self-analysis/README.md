## Why this is the point

Every rule below was reconstructed *from the repo's own history* — none of it is
written down as guidance anywhere in the source, and each links back to the commit
that earns it:

> **Do not pass git refs positionally without `--end-of-options` and validation; an
> option-like ref is a real arbitrary-file-write, not just a foot-gun.** …
> `plan --branch '--output=/path'` was parsed by git log as an option and wrote an
> arbitrary file (confirmed). `[observed]` *(61ca52e)*

> **Do not split git-log records on a byte that can occur in a commit body; it lets a
> hostile body crash the whole analysis.** … Fixed by NUL (`%x00`) separation plus
> greedy body parsing. `[observed]` *(0a4c5ac)*

> **Do not assume a line-count cap bounds diff size; one minified line can be
> megabytes.** `[observed]` *(1a49ab7)*

That's the tacit knowledge a senior engineer carries and a new agent doesn't — now
prescriptive, evidence-linked, and honest about what's *observed* vs merely *inferred*.

> **Note:** this is a point-in-time snapshot, not regenerated on every commit, so it
> drifts from the current source. Run the tool yourself for up-to-date memory.
